### 2. Duplicate Code: Section Color Definitions

**Locations:**
- `components/layout/SectionNavCard.tsx` (lines 11-18)
- `components/navigation/ContentsPanel.tsx` (lines 15-22)
- `components/ui/StatusPill.tsx` (lines 12-19)
- `lib/constants.ts` (lines 4-11)

**Issue:** The `sectionColors` object is defined independently in 4 different files with slightly different structures and fallback values.

```typescript
// SectionNavCard.tsx
const sectionColors: Record<string, { outer: string; inner: string }> = {
  'website-rc': { outer: '#f4b98c', inner: '#dc6b14' },
  // ...
};

// StatusPill.tsx
const sectionPillColors: Record<string, { dotColor: string; bgColor: string }> = {
  'website-rc': { dotColor: '#ED8A40', bgColor: '#FFE9D8' },
  // ...
};
```

**Impact:**
- Color inconsistencies across components
- Maintenance burden when colors change
- Different fallback defaults (e.g., `#e0e0e0` vs `#353535`)

**Recommendation:** Consolidate all section color definitions in `lib/constants.ts` with a unified structure.

## Medium Priority

### 4. Accessibility Concerns

**Location:** Audio player components

**Issues:**

a) **Click handlers on non-interactive elements** - Progress bars use `<div onClick>` instead of proper slider elements:
   - `components/audio/AudioPlayerCard.tsx` (lines 202, 287)
   - `components/feature/AudioDigestPlayer.tsx` (line 166)

b) **Limited ARIA attributes** - Only 11 ARIA attributes found across entire codebase. Audio players have minimal accessibility support:
   - Progress bars lack `aria-valuenow`, `aria-valuemin`, `aria-valuemax` (AudioPlayerCard)
   - Missing keyboard navigation for skip/seek controls

**Recommendation:**
- Replace progress bar divs with `<input type="range">` or proper ARIA slider implementations
- Add keyboard handlers for skip controls
- Ensure all interactive elements have appropriate ARIA labels

### 5. Native `<img>` Tags Instead of Next.js Image

**Locations:**
- `app/[edition]/audio-digest/page.tsx` (line 132)
- `app/[edition]/page.tsx` (line 41)
- `components/layout/SectionNavCard.tsx` (line 67)
- `components/navigation/SiteHeader.tsx` (line 41)
- `components/navigation/MobileNav.tsx` (lines 60, 110)
- `components/navigation/Footer.tsx` (lines 33, 42)
- `components/audio/AudioPlayerCard.tsx` (line 315)

**Issue:** 9 instances of native `<img>` tags bypass Next.js Image optimization.

**Impact:**
- Missed automatic image optimization (WebP conversion, resizing)
- No lazy loading by default
- Larger page payloads

**Note:** `components/feature/FeatureImage.tsx` correctly uses `next/image`.

**Recommendation:** Replace `<img>` with `<Image>` from `next/image`, especially for logos and content images.

## Patterns to Watch

1. **Client Component Proliferation** - 11 `'use client'` components. Ensure server components are preferred where possible.

2. **Inline Styles** - Significant use of inline `style={}` props. Consider moving to Tailwind classes or CSS variables for consistency.

3. **JSON Content Loading** - All content loaded via `fs.readFileSync` at build time. This is appropriate for static generation but watch for performance with many editions.
